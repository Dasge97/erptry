# Quality Gates

## Minimos para avanzar

- nombres consistentes;
- documentacion principal actualizada;
- contrato del modulo afectado identificado;
- tareas y estado sincronizados;
- deuda tecnica visible registrada si no se resuelve;
- sin secretos ni ficheros locales comprometidos.

## Minimos para cerrar una fase

- revision cruzada propia hecha;
- backlog actualizado;
- riesgos abiertos visibles;
- base entendible por otro desarrollador;
- si existe codigo, lint, typecheck y pruebas minimas en verde o fallo documentado.
