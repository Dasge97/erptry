# Skill: git workflow profesional

## Objetivo

Versionar de forma limpia, segura y preparada para publicacion futura.

## Cuando usarla

- antes de commit;
- antes de push;
- al preparar una fase estable.

## Reglas

- revisar `git status`;
- comprobar `.gitignore`;
- no subir secretos ni documentacion operativa sensible del servidor;
- usar mensajes de commit claros en espanol;
- hacer push solo cuando lo pida el usuario.

## Checklist

- cambios pertinentes identificados;
- secretos excluidos;
- documentacion sincronizada;
- rama y remoto revisados.

## Errores a evitar

- commits de ruido;
- mezclar cambios inconexos;
- asumir que push implica deployment.
